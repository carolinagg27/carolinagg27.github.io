  • EL MUNDO NO ES COLOR DE ROSA

    • Ada Lovelace desempeñó un papel significativo en el empoderamiento femenino, aunque su impacto no siempre se ha reconocido de inmediato. Aquí te detallo algunas formas en que su vida y trabajo contribuyeron a este objetivo: Lovelace ha llegado a simbolizar el empoderamiento femenino en la tecnología, inspirando eventos y programas educativos destinados a fomentar la participación de las mujeres en programación y ciencias de la computación. Su figura es utilizada como un emblema de lo que las mujeres pueden lograr en estos campos.

    Lovelace es reconocida por su visión de la máquina analítica no solo como una calculadora, sino como un dispositivo capaz de realizar cualquier tipo de cálculo que pudiera ser descrito mediante instrucciones. Escribió notas extensas sobre la máquina, y en estas notas, describió un algoritmo para calcular los números de Bernoulli, que se considera el primer algoritmo destinado a ser procesado por una máquina. Por esto, se le atribuye el título de la primera programadora de computadoras.

    1. A continuación, se explicará el concepto de branches:

    ¿Qué es una rama (branch) y por qué utilizarla?

    Una rama es una versión paralela de tu proyecto. Te permite trabajar en nuevas características, correcciones de errores o cualquier modificación sin afectar la rama principal (usualmente llamada main o master).

    Las ramas se utilizan en los siguientes casos:

    • Desarrollo de nuevas características: Crear una rama específica para una nueva función permite trabajar en ella sin romper la versión estable.

    • Correcciones de errores: Si encuentras un error, puedes crear una rama para corregirlo sin afectar el desarrollo en la rama principal.

    • Colaboración: Múltiples desarrolladores pueden trabajar en diferentes ramas y luego fusionar los cambios al proyecto principal.



    Primero se creará un archivo en el repositorio con nombre README.md con cualquier contenido, esto con el fin de crear un archivo inicial en nuestro repositorio:



    Posteriormente en la consola de git se agrega este archivo con el comando “git add README.md”. El comando “git add” se utiliza en Git para agregar cambios en tu directorio de trabajo al área de preparación (staging area). Este es un paso crucial en el flujo de trabajo de Git, ya que te permite seleccionar qué cambios deseas incluir en tu próximo commit.



    A continuación se ejecuta el comando git commit -m “Se agrega el archivo README.md”. El comando “git commit” se utiliza en Git para guardar los cambios en el área de preparación (staging area) en el historial del repositorio. Este comando es fundamental en el flujo de trabajo de Git, ya que crea un "snapshot" (instantánea) de los archivos y cambios que has preparado con git add.



    Para crear una rama se debe ejecutar el comando git Branch <nombre-rama>, por ejemplo: “git branch develop”. Este comando simplemente crea la rama, pero no cambia a ella.



    Se puede constatar la creación de la rama con el comando “git branch -a”, en este podemos apreciar que aún seguimos en la rama master, a pesar de que fue creada la rama develop.



    Para empezar a trabajar en la nueva rama, debes cambiar a ella usando el comando “git checkout develop”. Ahora estás trabajando en esa rama y cualquier cambio que hagas no afectará la rama principal.
